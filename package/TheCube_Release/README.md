# 认证器盒子PC Tool

## 简介

认证器盒子PC Tool是一个用于管理认证器和设备认证的桌面应用程序。该工具提供了图形化界面，支持设备自动识别、状态监控和批量认证功能。

## 功能特性

- **设备自动识别**: 自动识别连接的认证器和待认证设备
- **实时状态监控**: 定期刷新设备状态信息
- **认证器管理**: 支持认证器的锁定、解锁、激活和配置操作
- **设备认证**: 支持单个设备和批量设备认证
- **诊断功能**: 支持Token记录、TA诊断信息、认证记录的获取和导出 ⭐ 新增
- **WiFi管理**: 支持设备WiFi连接和断开操作
- **配置管理**: 支持配置文件的加载、导出和管理
- **日志管理**: 完整的日志记录和查看功能
- **用户友好界面**: 直观的GUI界面，操作简单

## 系统要求

- Python 3.6或更高版本
- ADB (Android Debug Bridge)
- USB连接的认证器和待认证设备

## 安装和运行

### 方法1: 使用启动脚本（推荐）

1. 双击 `start.bat` 文件启动程序
2. 如果Python未安装，脚本会提示安装

1. 打开命令提示符或PowerShell
2. 进入程序目录

## 使用方法

### 1. 连接设备

- 将认证器和待认证设备通过USB连接到PC
- 确保设备已开启ADB调试模式
- 程序会自动检测并分类设备

### 2. 查看认证器信息

- 认证器信息区域显示所有连接的认证器状态
- 包括过期时间、剩余设备数、已激活设备数等
- 不同颜色表示不同的过期状态

### 3. 连接WIFI

- 通过工具->认证器链接WIFI 以链接WIFI
  - SSID为WIFI名称
  - 密码为WIFI密码
  - 检查Snapshot栏，确认time_status是否为Ready，等待Ready后再进行使用

### 4. 认证设备

- **进行设备认证要求链接WIFI**

#### 单个设备认证

- 在待认证设备列表中双击设备
- 或选择设备后点击"认证"按钮
- 选择要使用的认证器
- 系统会自动完成认证流程

#### 批量认证

- 点击"一键认证全部"按钮
- 选择认证器
- 系统会自动认证所有未认证设备

### 5. 认证器操作

通过工具菜单可以执行：

- **锁定认证器**:
  - **需要链接WIFI**
  - 认证器出现异常时或需要寄回时，需要ATC提供token将认证器进行锁定，锁定期间无法进行认证/激活操作
- **解锁认证器**:
  - **需要链接WIFI**
  - 由ATC对认证器进行解锁，认证器将重新可以进行认证操作
- **激活认证器**:
  - **需要链接WIFI**
  - 激活认证器操作
  - Counter/Expire Date信息将被重置

### 激活码申请流程

- 抓取如下诊断信息，交由Autochips执行归档(**请不要尝试修改诊断信息，会导致ATC认证失败**)
  - 工具 -> 诊断日志 -> 获取Token记录
  - 工具 -> 诊断日志 -> 获取TA诊断信息
  - 工具 -> 诊断日志 -> 获取认证记录
- 联系对接人或邮件发送至Autochips, 包含如下信息:
  - authenticator_id
  - 诊断信息
  - 需要激活设备数
  - 最迟可激活时间(<= 72H)，激活码将在该时间段生效，默认最迟可激活时间为激活码产生24H内
  - 设备预计有效期 (默认为6个月内有效)
- Autochips将产生激活码
  - 对诊断信息进行认证以及归档
  - 产生激活码，告知激活码，可激活AC8267设备数, 过期时间

## 故障排除

### 常见问题

1. **设备未识别**
   - 检查USB连接
   - 确认设备ADB调试已开启
   - 检查ADB驱动是否正确安装

2. **无法进行认证**
   - 通过工具->认证器链接WIFI 以链接WIFI
     - SSID为WIFI名称
     - 密码为WIFI密码
     - 检查Snapshot栏，确认time_status是否为Ready，等待Ready后再进行使用
   - 确认认证器Counter是否为空
   - 确认认证器是否过期

3. **程序无法启动**
   - 检查Python版本（需要3.6+）
   - 检查依赖是否安装完整
   - 查看日志文件获取详细错误信息

4. **认证失败**
   - 确认认证器状态正常
   - 检查设备连接状态
   - 查看日志了解具体错误原因

### 日志查看

- 工具日志
  - 通过菜单"文件" -> "查看日志"查看运行日志
  - 日志文件位于`logs/cbss_tool.log`
  - 可以通过配置调整日志级别和路径
- ATC Log
  - 通过adb pull /data/debuglogger将日志抓出
- 认证器诊断信息
  - 工具 -> 诊断日志 抓取诊断信息
